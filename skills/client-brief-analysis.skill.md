# Client Brief Analysis Skill

## Purpose

Convert a client brief into clear project memory and action-ready direction.

## When To Use

Use when starting a project, receiving discovery notes, or reviewing unclear client input.

## Required Inputs

- Client brief or notes.
- Known goals, audience, offer, and constraints.
- Any deadlines or approval requirements.

## Process

1. Extract goals, audience, offer, deliverables, constraints, and success criteria.
2. Separate facts from assumptions.
3. Identify missing information and risks.
4. Recommend immediate next actions.
5. Update or draft `PROJECT_BRIEF.md` and `NEXT_ACTIONS.md`.

## Output Format

- Brief summary.
- Extracted facts.
- Assumptions.
- Open questions.
- Recommended next actions.

## Risks

- Treating vague client statements as requirements.
- Missing approval constraints.
- Creating scope before the client confirms it.
