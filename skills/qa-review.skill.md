# QA Review Skill

## Purpose

Review completed work against requirements, quality standards, and risks.

## When To Use

Use before approving, publishing, handing off, or closing a task.

## Required Inputs

- Original request.
- Acceptance criteria.
- Changed files or deliverable.
- Relevant project rules.

## Process

1. Compare output against the request.
2. Check completeness, clarity, consistency, and constraints.
3. Identify defects, risks, missing checks, and open questions.
4. Recommend fixes or approval.
5. Update issue or next-action memory if needed.

## Output Format

- Review status.
- Findings by severity.
- Missing verification.
- Recommended fixes.
- Approval recommendation.

## Risks

- Reviewing without the original request.
- Treating unverified work as complete.
- Failing to log unresolved issues.
