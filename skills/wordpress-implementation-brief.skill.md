# WordPress Implementation Brief Skill

## Purpose

Prepare clear, low-cost WordPress implementation instructions.

## When To Use

Use when a page, feature, content update, or CMS task needs to be implemented in WordPress.

## Required Inputs

- Approved scope.
- Page or feature requirements.
- Theme, builder, plugin, or hosting constraints if known.
- Content and assets.

## Process

1. Summarize the requested implementation.
2. List affected pages, templates, plugins, or settings.
3. Define step-by-step implementation actions.
4. Include verification steps and rollback notes.
5. Flag approval needs and risks.

## Output Format

- Implementation objective.
- Affected areas.
- Step-by-step instructions.
- Verification checklist.
- Risks and approval needs.

## Risks

- Assuming paid plugins.
- Changing production without approval.
- Ignoring backups, staging, or rollback.
