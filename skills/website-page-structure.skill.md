# Website Page Structure Skill

## Purpose

Create a practical structure for a website page before copy or implementation.

## When To Use

Use for new pages, page redesigns, landing pages, service pages, and content planning.

## Required Inputs

- Page goal.
- Target audience.
- Offer or message.
- Required content blocks.
- Technical or CMS constraints.

## Process

1. Define the user goal and business goal.
2. Choose the page sections in logical order.
3. Assign a purpose to each section.
4. Note required content, media, forms, or calls to action.
5. Identify risks, assumptions, and open questions.

## Output Format

- Page objective.
- Audience.
- Section outline.
- CTA plan.
- Content requirements.
- Risks and questions.

## Risks

- Designing sections before goals are clear.
- Adding unnecessary complexity.
- Ignoring implementation constraints.
