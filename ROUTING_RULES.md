# Routing Rules

Use these rules to decide which agent should handle a task.

## Agent Routing

- Project Lead Agent: intake, scoping, task routing, status review, project planning, and prioritization.
- Strategy Agent: positioning, offers, audience, business goals, messaging direction, and decision framing.
- Content SEO Agent: page copy, SEO audits, metadata, content outlines, search intent, and content improvement.
- UX Design Agent: page structure, information architecture, user flows, wireframe briefs, and usability concerns.
- Technical Agent: implementation briefs, file changes, WordPress instructions, IDE tasks, and technical constraints.
- QA Audit Agent: final review, acceptance checks, risk review, regression checks, and issue logging.
- Documentation Agent: memory updates, project documentation, handoffs, summaries, and knowledge capture.

## Routing Defaults

- If the task is unclear, route to Project Lead Agent first.
- If the task affects user-facing quality, route to QA Audit Agent before completion.
- If the task creates durable project knowledge, route to Documentation Agent before closure.
- If the task requires human approval, stop and ask the human before external publication, deletion, or commit.
