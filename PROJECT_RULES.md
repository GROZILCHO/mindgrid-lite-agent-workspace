# Project Rules

These rules keep the workspace low-cost, transparent, and human-controlled.

## Safety Rules

- Do not assume paid OpenAI Skills, Claude, custom backend, vector database, or automation platform.
- Do not install packages unless the human explicitly approves.
- Do not modify Git configuration unless the human explicitly approves.
- Do not commit, push, publish, send, or deploy without human approval.
- Do not delete project memory or overwrite important context without review.
- Do not create project-specific folders until a real project is approved.

## Operating Rules

- ChatGPT Project is the command center, not the source of truth.
- Markdown files are the shared memory and operational state.
- VS Code, Codex, or another IDE agent is the file execution environment.
- Git is the change history and review layer.
- The human user is the approval gatekeeper.

## Quality Rules

- Keep content practical, concise, and operational.
- Prefer explicit assumptions over hidden assumptions.
- Record decisions and next actions in Markdown.
- Use templates and workflows before inventing new structures.
