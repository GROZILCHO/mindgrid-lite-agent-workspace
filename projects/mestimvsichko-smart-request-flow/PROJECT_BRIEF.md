# Project Brief

## Project Name

Mestimvsichko Smart Request Flow

## Purpose

Create a practical, structured request intake flow for Mestimvsichko.bg so customers can submit clearer moving, transport, loading/unloading, storage, and related service inquiries.

The flow must improve request quality without promising automatic pricing, confirmed booking, online payment, AI automation, GPS tracking, or full CRM behavior.

## Business Context

Mestimvsichko.bg is a WordPress-based service website serving private clients and small businesses in the Varna region.

Phase 1 remains the website redesign and practical improvement of request intake. The Smart Request Flow is the MVP for a more structured inquiry process inside that broader direction.

## Target Users

- Private clients planning a move, transport job, loading/unloading, storage, or related service.
- Small businesses that need practical logistics support.
- Internal Mestimvsichko staff who need clearer request details before contacting the customer.

## MVP Scope

- Service type selection.
- Addresses and access conditions.
- Items, inventory, or description of what needs to be moved.
- Photos and additional services.
- Contact details and request submission.
- Bulgarian-language user experience.
- WordPress-first implementation direction.
- Clear request-submission language, not booking or final-offer language.
- Limited and safe image upload expectations, such as up to 5 images in JPG, PNG, or WebP with reasonable file size limits.

## Out Of Scope

- Automatic price calculation.
- Online payment.
- Reservation or calendar booking.
- GPS or live tracking.
- AI automation promises.
- Full CRM automation.
- Misleading automatic offer language.

## Strategic Opportunity

If the MVP works for Mestimvsichko, the Smart Request Flow may become a reusable service/product pattern for small service businesses that need better inquiry intake without expensive custom software.

## Related Technical Direction

A custom WordPress plugin named MindGrid Request System exists as a related technical direction. This project memory focuses on business, UX, and product coordination for the Mestimvsichko Smart Request Flow, not implementation work.

## Success Criteria

- Customers understand they are submitting a request, not making a confirmed booking.
- Mestimvsichko receives more complete and useful inquiry details.
- Phase 1 remains practical and affordable.
- Future reusable-product potential is preserved without adding unnecessary MVP complexity.
