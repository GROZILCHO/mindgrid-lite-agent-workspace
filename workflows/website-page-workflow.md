# Website Page Workflow

Use this workflow for creating or improving a website page.

## Steps

1. Project Lead confirms page goal, audience, constraints, and acceptance criteria.
2. Strategy Agent confirms positioning and message direction if needed.
3. UX Design Agent drafts page structure and section purpose.
4. Content SEO Agent drafts or audits copy and metadata.
5. Technical Agent prepares implementation notes or makes approved file changes.
6. QA Audit Agent reviews the page against goals and constraints.
7. Documentation Agent updates memory and next actions.

## Required Memory Updates

- `TASKS.md`
- `CURRENT_STATUS.md`
- `ISSUES_LOG.md` if risks are found
- `NEXT_ACTIONS.md`
- `AGENT_HANDOFF.md` if work continues
