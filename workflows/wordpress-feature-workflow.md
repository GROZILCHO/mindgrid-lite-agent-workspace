# WordPress Feature Workflow

Use this workflow for WordPress pages, features, plugin settings, or CMS changes.

## Steps

1. Project Lead confirms scope, environment, approval needs, and risk level.
2. Strategy or UX Design Agent clarifies user value and expected behavior if needed.
3. Technical Agent prepares a WordPress implementation brief.
4. Human approves any production, plugin, theme, or settings change.
5. Technical Agent performs approved implementation or produces instructions.
6. QA Audit Agent checks behavior, content, responsiveness, and rollback notes.
7. Documentation Agent updates status, decisions, issues, and next actions.

## Required Memory Updates

- `DECISIONS_LOG.md` for approved technical choices
- `TASKS.md`
- `ISSUES_LOG.md`
- `NEXT_ACTIONS.md`
