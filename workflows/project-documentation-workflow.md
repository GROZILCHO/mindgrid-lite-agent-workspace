# Project Documentation Workflow

Use this workflow to create or refresh project memory.

## Steps

1. Project Lead identifies which memory files are missing, stale, or inconsistent.
2. Documentation Agent reviews available notes, outputs, and decisions.
3. Documentation Agent updates the relevant memory files.
4. QA Audit Agent checks for contradictions, unsupported claims, and missing next actions.
5. Project Lead confirms the current status and next task.

## Required Memory Updates

- `PROJECT_BRIEF.md`
- `CURRENT_STATUS.md`
- `DECISIONS_LOG.md`
- `TASKS.md`
- `ISSUES_LOG.md`
- `NEXT_ACTIONS.md`
- `AGENT_HANDOFF.md`
