# Standard Workflow

Use this workflow for most work unless a specific workflow file applies.

## Flow

1. User Request
   - The human defines the need, constraint, or desired output.
   - The human remains the approval gatekeeper.

2. Project Lead
   - Clarifies scope.
   - Checks project memory.
   - Selects the right specialist agent.
   - Defines expected output and acceptance criteria.

3. Specialist Agent
   - Performs the assigned work using relevant memory, templates, and skills.
   - Writes or edits only approved files.
   - Records assumptions, decisions, and unresolved questions.

4. QA
   - Reviews the output against the request, project rules, and acceptance criteria.
   - Logs issues, risks, and recommended fixes.

5. Documentation
   - Updates project memory files.
   - Captures decisions, current status, and handoff notes.

6. Next Action
   - Updates `NEXT_ACTIONS.md`.
   - Makes the next human approval or agent task explicit.

## Default Completion Standard

A task is complete only when the output exists, relevant memory is updated, risks are recorded, and the next action is clear.
